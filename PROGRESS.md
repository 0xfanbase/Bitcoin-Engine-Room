### 2026-07-11 — Info-toggle hover glitch fixed, expand-all button added, UI/UX audit workflow

Owner report: hovering the "i" next to Price (USD) triggered an unending open/close flicker, plus a request to switch info-toggles to a click-driven "expand all descriptions" button and do a full UI/UX audit of every button/toggle.

- **Root cause of the hover glitch, confirmed by measurement, not guesswork**: `.masthead-stat` was a shrink-to-fit flex item with its `.info-panel` (max-width 220px) nested directly inside. Opening the panel widened the item's box past its collapsed label/value width; at several common viewport widths (verified 414px/480px/700px via Playwright bounding-box diffs) that extra width pushed the 3-stat row past its flex-wrap threshold, reflowing the row and shifting the "i" button ~100px sideways out from under the pointer -- which fired `mouseleave` (close) then `mouseenter` again (open) forever.
- **Fix**: `assets/info.js` now reveals on click only (hover/focus auto-reveal removed entirely -- touch and mouse users get identical behavior). `.masthead-stat` got a fixed `width: 205px` so opening its panel only ever grows height, never width. Re-verified zero shift across 13 widths from 320px-1440px.
- **New `#expand-all-toggle`** button at the top of the masthead opens (or closes) all 15 button-driven `.info-panel`s *and* the 3 native `<details class="info-disclosure">` section explainers (they carry the same "i" bullet visually) in one action, flipping its own label/`aria-pressed` to match.
- **Full end-to-end UI/UX audit**: ran a `Workflow` (5 parallel finder agents across chart controls, the info-toggle system, health/audit panels, responsive layout, and JS-correctness/accessibility, each browser-driving Playwright against a local server, then every finding adversarially verified by a second agent before counting) rather than a single read-through, since the ask was explicitly end-to-end. All 8 candidate findings survived verification (an unusually high confirm rate, but each carried its own independent live repro, not just a code-reading argument) and all 7 distinct bugs (one finding was a duplicate angle on the same aria-pressed gap) got fixed:
  1. `charts.js`'s Power Law timeframe chips (1Y/4Y/ALL) threw an uncaught TypeError if clicked before `data/models.json` finished loading (a real race condition, not just a hard-failure edge case) -- `renderPowerLaw()` dereferenced `modelsDoc.power_law` with no guard, unlike `renderAll()`'s `if (!modelsDoc) return;`. Added the same guard.
  2. Those same chip buttons never got `aria-pressed` reflecting the active one (only a CSS class changed) -- added `aria-pressed` to the markup and the click handler, matching the pattern `#expand-all-toggle` already used.
  3. The "overdue snapshot age" red-flag styling was dead on arrival: `health.js` added only the class `is-overdue` to the age span, but the only CSS rule was the compound `.audit-generated-at.is-overdue` (a *different* element -- the audit panel's own timestamp). De-compounded the selector to standalone `.is-overdue`.
  4. `renderAuditPanel()` threw if `data/audit/latest.json` loaded successfully but its body omitted `findings` (partial write / schema drift) -- added the same `|| []` fallback `renderHealthGrid()` already uses for `health.metrics`.
  5. At exactly 320px viewport width, the mobile 2-column gauges grid (`repeat(2, 1fr)`) had no minimum-width floor, so a plausible 4-digit difficulty reading (Bitcoin's real difficulty only grows) blew the grid -- and the whole page, since `html` has `overflow-x: hidden` -- past the viewport with no way to scroll to the clipped pixels. Fixed with `minmax(0, 1fr)` plus `overflow-wrap: anywhere` on `.gauge-value`/`.gauge-sub` so a long token wraps inside its card instead of forcing it wider.
  6. `#expand-all-toggle` tracked "all expanded" via its own last-set `aria-pressed` rather than actual panel state, so manually closing one panel after using it desynced the button -- a second click would then close everything instead of finishing the job. Now computed live from whether anything is still closed.
  7. No `aria-live` region anywhere on the page, so the site's one designed theatrical event (a new block arriving) was silent to screen readers even though it visibly updates the odometer. Added `aria-live="polite" aria-atomic="true"` to `#odometer` specifically (not the per-second-ticking captions, which correctly stay non-live).
  8. Removed two dead formatters (`BER.formatCompact`, `BER.formatDate`) with zero call sites anywhere in the codebase.
- Every fix re-verified live (Playwright, not just re-reading the diff): the modelsDoc race no longer throws, chips get correct `aria-pressed`, the overdue span now resolves `--fail` red, a findings-less audit.json no longer throws and shows "No findings", 320px stays within viewport with stressed gauge values (screenshotted: values wrap cleanly inside their cards), expand-all correctly finishes opening the one remaining panel instead of closing everything, and the odometer carries the right ARIA attributes. `pytest pipeline/tests -v`: 115/115 passing throughout (frontend-only change, but re-ran to be sure).
